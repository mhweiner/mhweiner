Marvel.comVisit Website

Marvel.com broke new ground in 2013 to become one of the first large-scale, media-rich, high-traffic fully-responsive websites. The website underwent a complete overhaul, and expanded its role, unifying all of Marvel's business verticals.

To make the challenge even more difficult, the designers wanted a bold, "on-brand" design, instead of a compromised watered-down experience typical of responsive websites at the time. We were given a 9 month deadline, and to top it off, we needed to support IE8!

There were many technical challenges. The biggest ones were:

  • Payload size of CSS, HTML, Javascript, hi-res images, and other assets.
  • Older mobile connection speeds in 2013 (3G at best).
  • Performance, especially on mobile devices. Double especially older mobile devices.
  • The scope of the website, deadline, and lack of examples. We were breaking new ground.

Involvement

I led R&D, front-end architecture, and front-end development. Worked closely with design and backend teams.

  • Worked with team to create solutions to major challenges, such as payload size, performance, legacy, maint. costs
  • Invented CSS variant export technique, co-created with multiple team members
  • Led in key initiatives to help ensure the front-end team can deliver by tight deadline
  • Led in setting standards, methodologies and conventions to ensure a smooth and efficient process, such as the Responsive Module Inventory
  • Built a custom automated test program for responsive websites using multi-process PHP and image diff tools

Technology

  • HTML5/CSS3/Javascript/ZeptoJS
  • Newfangled "device agnostic feature based responsive design" methodology
  • Custom sass extension CSS Variant Export Script written in Ruby
